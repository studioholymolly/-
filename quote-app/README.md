# 홀리몰리 견적서 · 계약서 (독립 앱)

스튜디오 홀리몰리 운영 대시보드(`holymolly-ops.vercel.app`)의 **견적·계약서 기능**을 분리해 만든 독립 웹앱입니다.
기존 대시보드는 배포된 결과물만 남아 있고 소스가 없어, 배포 번들에서 견적·계약 기능을 복원한 뒤 아래 요청 사항을 반영했습니다.

## 이번 변경 — 견적서 항목 구조

- 견적서 항목을 **두 구역으로 분리**: **① 포토그래퍼 촬영비**(권장 5개) / **② 스타일리스트 · 외주**(권장 5개), 최대 10개.
- 각 항목에 `촬영 / 외주` 구분 토글 추가. 프리셋 칩과 항목의 이름·구분에 따라 자동으로 두 구역에 배치됩니다.
- 인쇄용 견적서(A4)도 두 구역으로 나뉘어 각 5줄(합계 10줄)로 표시됩니다.

## 데이터 — 기존 대시보드와 공유

- 같은 Supabase(`quotes` 테이블)에 저장하므로, 여기서 저장한 견적/계약서는 **대시보드 '발급 내역'에도 그대로 표시**됩니다.
- 고객사 목록은 대시보드의 `clients`에서 읽어옵니다.
- 프리셋(단가표) 편집은 이 앱의 **브라우저 로컬 저장소에만** 반영되어, 대시보드 공용 설정(`app_config`)은 변경하지 않습니다. (기존 대시보드에 영향 0)

## 기술 스택

- React 18 + Vite, `@supabase/supabase-js`. 인쇄/PDF는 브라우저 인쇄(`window.print()`) 사용.
- 로그인은 기존 대시보드와 **동일한 Supabase 계정**으로 진행합니다.

## 로컬 실행

```bash
npm install
cp .env.example .env   # 값 입력 (아래 참고)
npm run dev
```

## 배포 (Vercel)

1. 이 저장소를 Vercel에 Import.
2. **Root Directory** 를 `quote-app` 로 지정 (Framework: Vite 자동 감지).
3. 환경변수 2개 입력 — `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`
   (값은 기존 홀리몰리 대시보드와 동일합니다).
4. Deploy.

## 환경변수

| 이름 | 설명 |
|---|---|
| `VITE_SUPABASE_URL` | Supabase 프로젝트 URL (기존 대시보드와 동일) |
| `VITE_SUPABASE_ANON_KEY` | Supabase anon key (기존 대시보드와 동일, 공개용 키) |
